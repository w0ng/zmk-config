When creating git commits, always use Conventional Commits format:

Format: `<type>(<scope>): <subject>`

Types:
- feat: New feature
- fix: Bug fix
- docs: Documentation changes
- style: Code style changes (formatting, etc.)
- refactor: Code refactoring
- test: Adding or updating tests
- chore: Maintenance tasks

Examples:
- `feat(keymap): add symbol layer customization`
- `fix(config): correct bluetooth device name`
- `docs(readme): update hardware section`
- `refactor(symbols): reorganize bracket positions`
